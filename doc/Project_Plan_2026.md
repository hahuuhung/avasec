# Kế hoạch Quản lý Dự án SysAnti (Project Management Plan)

## 1. Thông tin Dự án

* **Tên dự án:** SysAnti - Ultimate System Care
* **Mục tiêu:** Xây dựng phần mềm tối ưu hóa và bảo mật Windows số 1 tại Việt Nam.
* **Ngân sách:** [Điền ngân sách]
* **Thời gian:** Q1 2026 - Q4 2026

## 2. Các Mốc Quan trọng (Milestones)

| Phase | Thời gian | Mục tiêu | Trạng thái |
|-------|-----------|----------|------------|
| **Phase 1: MVP** | 01/2026 - 02/2026 | Hoàn thiện UI/UX, Core Features (Clean, Boost). | ✅ Hoàn thành |
| **Phase 2: Beta** | 03/2026 - 04/2026 | Phát hành thử nghiệm, fix bugs, tối ưu hiệu năng. | 🚧 Đang thực hiện |
| **Phase 3: Launch** | 05/2026 | Ra mắt chính thức v1.0, Marketing, Bán License. | 📅 Lên kế hoạch |
| **Phase 4: Ecosystem**| Q3-Q4 2026 | Mobile App, Server update, Chat Support AI. | 📅 Lên kế hoạch |

## 3. Quy trình Làm việc (Workflow)

### 3.1. Phương pháp: Agile / Scrum

* **Sprint:** 2 tuần.
* **Daily Standup:** 15 phút mỗi sáng (9:00 AM).
* **Weekly Review:** Chiều thứ 6 hàng tuần.

### 3.2. Công cụ Quản lý

* **Source Code:** GitHub / GitLab.
* **Task Management:** Jira / Trello.
* **Documentation:** Folder `Docs` trong source code + Wiki nội bộ.
* **Communication:** Slack / Discord.

## 4. Quản lý Rủi ro (Risk Management)

| Rủi ro | Mức độ | Phương án Giải quyết |
|--------|--------|----------------------|
| Thay đổi requirements | Cao | Chốt spec kỹ đầu mỗi Sprint. Có Change Request process. |
| Technical Debt | Trung bình | Dành 20% thời gian mỗi sprint để refactor code. |
| Nhân sự | Cao | Cross-training, tài liệu hóa kỹ thuật đầy đủ (`Docs/For_Developers`). |
| Bảo mật | Rất cao | Audit code thường xuyên, thuê pentest trước khi Launch. |

## 5. Đội ngũ Dự án (Team Structure)

* **Project Manager (PM):** Quản lý tiến độ, nguồn lực.
* **Business Analyst (BA):** Lấy yêu cầu, viết document.
* **UI/UX Designer:** Thiết kế giao diện.
* **Dev Team:** Frontend (WPF), Backend (.NET API), System (C++).
* **QA/Tester:** Kiểm thử chât lượng.
