# 🧪 Hướng dẫn Chạy và Kiểm thử SysAnti

# SysAnti Run & Test Guide

> **Ngày cập nhật / Last Updated:** 2026-02-06 11:06  
> **Phiên bản / Version:** 1.0

Tài liệu này hướng dẫn cách build và chạy các thành phần của hệ thống SysAnti để kiểm tra giao diện và tính năng.

---

## 📋 Yêu cầu Hệ thống / System Requirements

- **.NET SDK 9.0** (Đã được cài đặt trên máy)
- **PowerShell** hoặc **Command Prompt**

---

## 🛠️ 1. Build Toàn bộ Dự án / Build All Projects

Trước khi chạy, hãy đảm bảo toàn bộ solution được build thành công để tránh lỗi thiếu file.

```powershell
# Di chuyển đến thư mục gốc của dự án (f:\VStudio\SysAnti)
# Chạy lệnh build
dotnet build
```

---

## 🚀 2. Chạy Giao diện Dashboard (WPF) / Run Dashboard UI

Đây là phần quan trọng nhất để xem giao diện Dashboard teal mới mà tôi vừa implement.

```powershell
# Chạy dự án UI
dotnet run --project SysAnti.UI\SysAnti.UI.csproj
```

**Lưu ý:**

- Khi chạy lệnh này, một cửa sổ Windows trắng mang tên "SysAnti - Dashboard" sẽ xuất hiện.
- Bạn có thể tương tác với các hiệu ứng hover trên các thẻ (cards) và nút bấm.

---

## 🌐 3. Chạy Backend API / Run Backend API

Nếu bạn muốn kiểm tra khả năng giám sát từ xa (remote monitoring).

```powershell
# Chạy dự án API
dotnet run --project src\Backend\SysAnti.MonitoringAPI\SysAnti.MonitoringAPI.csproj
```

**Truy cập Swagger:**
Sau khi API chạy, bạn có thể xem tài liệu API tại
`https://localhost:<PORT>/swagger` (Port thường là 5001 hoặc 7001, xem trong log khi chạy).

---

## 📡 4. Chạy Dịch vụ Báo cáo Trạng thái / Run Status Reporter

Thành phần này thu thập thông tin CPU/RAM và gửi lên API.

```powershell
# Hiện tại thành phần này được tích hợp trong UI hoặc chạy độc lập qua Console
# (Đang trong quá trình hoàn thiện tích hợp hoàn toàn)
```

---

## 🔍 5. Các kịch bản Kiểm thử / Test Scenarios

### 🟢 Giao diện (UI Testing)

1. **Kiểm tra Hover:** Rê chuột qua các thẻ CPU, RAM, Disk để thấy hiệu ứng đổ bóng thay đổi.
2. **Kiểm tra Quick Actions:** Rê chuột qua các icon 🧹, ⚡, 🛡️, 🚀 để thấy màu nền thay đổi sang xanh Teal sáng.
3. **Kiểm tra Resize:** Thử thay đổi kích thước cửa sổ để xem tính phản hồi (responsive).

### 🔵 Backend (API Testing)

1. Sử dụng Swagger để gọi thử endpoint `GET /api/StatusReporting` để xem danh sách thiết bị.

---

## ⚠️ Giải quyết Sự cố / Troubleshooting

- **Lỗi XAML:** Nếu gặp lỗi "XamlParseException", hãy thử chạy `dotnet clean` rồi build lại.
- **Port bận:** Nếu API không chạy được do port 5001 bị chiếm, hãy đổi port trong file `appsettings.json` của MonitoringAPI.
